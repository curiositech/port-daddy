pd spawn

Launch Ollama, Claude, Aider, and Gemini agents from one command. Port Daddy coordination — sessions, heartbeats, notes, salvage — is auto-wired.

pd spawn launches AI agents (Ollama, Claude API, Aider, Gemini, or any custom subprocess) with Port Daddy coordination pre-wired. The spawned agent is registered as a Port Daddy agent, sends heartbeats automatically, logs its responses as notes, and enters the salvage queue if it crashes.

What pd spawn Does

Without pd spawn, integrating an AI agent into Port Daddy requires manual calls: register the agent, start a session, send heartbeats in a loop, write notes, end the session on completion. pd spawn does all of this for you.

The agent you spawn becomes a full Port Daddy citizen: it appears in the dashboard, its output becomes queryable notes, other agents can pub/sub with it, and if it dies unexpectedly its last note becomes its salvage context.

Quick Start — Ollama

Spawn a local Ollama agent to perform a coding task:

# Prerequisite: Ollama must be running (ollama serve)
$ pd spawn \
    --backend ollama \
    --model llama3.2:8b \
    --identity myapp:coder \
    --purpose "Refactor auth module" \
    -- "Fix the login bug in src/auth.ts"

Agent spawned: agent-a1b2c3
Backend:  ollama (llama3.2:8b)
Identity: myapp:coder
Session:  sess-d4e5f6
Heartbeat: every 30s

[llama3.2:8b] Analyzing src/auth.ts...
[llama3.2:8b] Found issue: session token not invalidated on logout.
[llama3.2:8b] Fix: call destroySession() in the logout handler.
Agent done: sess-d4e5f6 ended

The -- separator marks the end of pd spawn flags. Everything after it is the prompt sent to the agent.

Backends

Five backends are supported. Each is tuned for its runtime:

# Ollama — local inference, uses HTTP to port 11434
$ pd spawn --backend ollama --model llama3.2:8b \
    --identity myapp:coder -- "Review src/auth.ts"

# Claude API — uses @anthropic-ai/sdk directly (not the CLI)
$ pd spawn --backend claude --model claude-haiku-4-5 \
    --identity myapp:reviewer -- "Review PR for security issues"

# Gemini — uses @google/generative-ai
$ pd spawn --backend gemini --model gemini-flash \
    --identity myapp:analyst -- "Summarize the test failures"

# Aider — git-native coding agent, wraps stdout as PD notes
$ pd spawn --backend aider --model gpt-4o \
    --identity myapp:writer -- src/auth.ts src/middleware.ts

# Custom — any subprocess, stdout becomes PD notes
$ pd spawn --backend custom \
    --identity myapp:worker -- python3 ./process.py

Coordination Auto-Wired

Every spawned agent automatically gets:

  1. An agent registration with the identity you specify
  2. A Port Daddy session opened with the purpose you specify
  3. Heartbeats sent every 30 seconds
  4. Each LLM response chunk added as an immutable note
  5. Session ended cleanly when the agent completes
  6. Resurrection queue entry if the agent crashes unexpectedly

Other agents can see spawned agents in pd salvage, read their notes, and pick up where they left off if they crash.

Managing Agents

# See all running spawned agents
$ pd spawned
AGENT ID      BACKEND   MODEL           IDENTITY       STATUS
agent-a1b2c3  ollama    llama3.2:8b     myapp:coder    running
agent-b2c3d4  claude    haiku-4-5       myapp:review   running

# Get JSON output for scripting
$ pd spawned --json

# Stop a specific agent
$ pd spawn kill agent-a1b2c3
Agent agent-a1b2c3 stopped. Session ended.

SDK Equivalent

import { PortDaddy } from 'port-daddy';

const pd = new PortDaddy();

// Spawn an Ollama agent
const agent = await pd.spawn({
  backend: 'ollama',
  model: 'llama3.2:8b',
  identity: 'myapp:coder',
  purpose: 'Refactor auth module',
  prompt: 'Fix the login bug in src/auth.ts',
});

// Stream output as it arrives
agent.on('output', (chunk) => console.log(chunk));
agent.on('done', () => console.log('Agent finished'));

// Or await completion
const result = await agent.wait();
console.log(result.notes);
