Harbor Services

Everything your fleet needs

From port assignment to fleet coordination, Port Daddy keeps your development vessels in safe waters.

Atomic Port Assignment

SQLite-backed assignments ensure zero collisions. Same vessel always gets the same berth, even after rough seas.

Ship's Registry

Semantic identities like fleet:vessel:route. Query with wildcards to find any ship in your harbor.

Fleet Deployment

port-daddy up launches your entire fleet. Dependency ordering, health signals, and ship-to-ship logs.

Deep Charting

Scans 5 fathoms deep to chart 60+ frameworks. Handles flotillas, workspaces, and nested harbors automatically.

Signal Flags

Pub/sub messaging between vessels. Real-time SSE for fleet coordination across agents.

Mooring Locks

Distributed locks prevent collisions at the dock. Exclusive access to shared cargo like databases.

Captain's Log

Works with Claude, Cursor, Windsurf, and 40+ AI agents. Every captain knows the fleet's status.

Harbor Master's Office

Web dashboard at localhost:9876. Monitor vessels, locks, and fleet activity in real-time.

Automatic Sweep

Tracks vessel IDs and clears dead ships from the registry. No ghost ships clogging your harbor.

Fleet Command

One command to deploy the fleet

Define your vessels in .portdaddyrc and run port-daddy up. Dependencies are resolved, health signals checked, and logs multiplexed with color-coded prefixes.

.portdaddyrc
CLI
SDK
{
  "fleet": "myapp",
  "vessels": {
    "api": {
      "cmd": "npm run dev:api",
      "port": 3001,
      "health": "/health"
    },
    "web": {
      "cmd": "npm run dev",
      "port": 3000,
      "needs": ["api"]
    },
    "worker": {
      "cmd": "npm run worker",
      "needs": ["api"],
      "noPort": true
    }
  }
}
# Chart the waters (auto-detect)
$ port-daddy scan

# Deploy the fleet
$ port-daddy up

# Deploy single vessel + dependencies
$ port-daddy up --service web

# Use git branch as route
$ port-daddy up --branch

# Signal other vessels
$ port-daddy pub build:api '{"status":"moored"}'
$ port-daddy sub build:*

# Mooring locks
$ port-daddy lock db-migrations && npm run migrate
$ port-daddy unlock db-migrations
import { PortDaddy } from 'port-daddy/client';

const harbor = new PortDaddy();

// Claim a berth
const { port } = await harbor.claim('myapp:api');

// Signal flags
await harbor.publish('fleet:ready', { vessel: 'api' });
const watch = harbor.subscribe('deployments');
watch.on('message', (signal) => console.log(signal));

// Mooring locks
await harbor.withLock('cargo-bay', async () => {
  await unloadCargo();
});
