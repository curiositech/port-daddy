Features

Everything you need for local development

From atomic port assignment to service orchestration, Port Daddy handles the complexity so you can focus on shipping.

Atomic Port Assignment

SQLite-backed ACID transactions ensure zero race conditions. Same project always gets the same port, even across restarts.

Semantic Identities

Use project:stack:context naming like myapp:api:feature-auth. Query with wildcards: myapp:*

Service Orchestration

port-daddy up starts your entire stack. Topological ordering, health checks, colored logs—like docker-compose for local dev.

Deep Auto-Detection

Scans 5 levels deep to find 60+ frameworks. Handles monorepos, npm workspaces, pnpm, and nested services automatically.

Pub/Sub Messaging

Agents signal each other when builds finish. Real-time SSE subscriptions for inter-service communication.

Distributed Locks

Prevent conflicting operations across agents. Take exclusive locks on databases, migrations, or any shared resource.

AI Agent Native

Works with Claude, Cursor, Windsurf, Cline, Aider, and 40+ agents. Published skill for automatic coordination.

Web Dashboard

Monitor services, locks, agents, and activity in real-time. Dark-themed UI at localhost:9876.

Auto-Cleanup

Tracks PIDs and automatically frees ports from crashed processes. No more forgotten assignments.

Semantic Identity

Named services, not magic numbers

Port Daddy uses a project:stack:context naming scheme that makes your services discoverable and queryable. No more remembering that the API is on port 3001.

  • ✓ myapp:api:main — the main API
  • ✓ myapp:frontend:feature-auth — branch-specific frontend
  • ✓ myapp:*:feature-auth — wildcard to find all services on a branch

Orchestration

One command to rule them all

Define your services in .portdaddyrc and run port-daddy up. Dependencies are resolved via topological sort, health checks ensure readiness, and logs are multiplexed with color-coded prefixes.

.portdaddyrc
CLI Usage
JavaScript SDK
{
  "project": "myapp",
  "services": {
    "api": {
      "cmd": "npm run dev:api",
      "port": 3001,
      "healthPath": "/health"
    },
    "frontend": {
      "cmd": "npm run dev -- --port ${PORT}",
      "port": 3000,
      "needs": ["api"]
    },
    "worker": {
      "cmd": "npm run worker",
      "needs": ["api"],
      "noPort": true
    }
  }
}
# Start everything with auto-detected config
$ port-daddy up

# Start one service and its dependencies
$ port-daddy up --service frontend

# Use git branch in identity
$ port-daddy up --branch

# Graceful shutdown
$ port-daddy down

# Publish/subscribe between agents
$ port-daddy pub build:api '{"status":"ready"}'
$ port-daddy sub build:*

# Distributed locks
$ port-daddy lock db-migrations && npm run migrate
$ port-daddy unlock db-migrations
import { PortDaddy } from 'port-daddy/client';

const pd = new PortDaddy();

// Claim a port
const { port } = await pd.claim('myapp:api');

// Pub/sub messaging
await pd.publish('builds', { status: 'complete' });
const sub = pd.subscribe('deployments');
sub.on('message', (data) => console.log(data));

// Distributed locks
await pd.withLock('deploy-prod', async () => {
  await deployToProduction();
});
