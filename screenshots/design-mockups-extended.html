Getting Started with Port Daddy

Learn how to claim stable ports, register DNS hostnames, and orchestrate your development stack with a single daemon.

Prerequisites

Port Daddy requires Node.js 18+ and runs on macOS, Linux, and WSL. Install it globally:

npm install -g port-daddy
pd start
pd status

Your First Port Claim

Port Daddy uses semantic identities in project:stack:context format. The same identity always maps to the same port:

pd claim myapp:api       # Always gets port 3101
pd claim myapp:api       # Idempotent — same port again
pd find myapp:api        # Shows port, status, uptime

const result = await pd.dnsRegister('myapp:api', { port: 3101 });
